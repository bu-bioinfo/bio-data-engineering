This is a **studio** course: class time is spent building, with staff circulating, over a
single **recurring dataset**. Almost every week you produce a working artifact, and the
artifacts chain together into a version-controlled, employer-facing **portfolio**.

Every artifact has the same three parts:

- **The deliverable** — working code / schema / config (machine-checkable; run the autograder
  yourself before submitting).
- **The rationale** — a short ADR-style design-decision document: what you chose, what you
  rejected, why.
- **The AI disclosure** — what tools you used, for what, where they were wrong, and how you
  verified.

The artifact sequence:

- **A0** — Environment + repo + "hello dataset" exploration.
- **A1** — Domain schema (LinkML) for the recurring dataset.
- **A2** — Ontology/identifier binding + semantic validation.
- **A3** — Three-way storage implementation + comparative analysis.
- **A4** — FAIR assessment + remediation plan for a real resource.
- **A5** — Validated ETL pipeline (raw → schema-conformant store).
- **A6** — Provenance/lineage layer over the A5 pipeline.
- **Capstone** — full stack for a self-chosen domain.

Two meta-logs run the whole term and are graded at midpoint and end: a cumulative **AI-use
disclosure log** and a **failure log**. LLM tools are assumed and expected
([AIAS](https://aiassessmentscale.com/) Level 4 for most artifacts): your rationale,
verification, disclosure, and failure log are the graded product.
